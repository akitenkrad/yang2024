# ユースケース

この OASIS 再現実装でできることと，詳細ドキュメントへの案内．

## 1. 情報拡散カスケードを観察する

小規模 X シミュレーションを実行し，カスケードが (単一ブロードキャストでなく) 多段に広がる様子を見る．フラグは [CLI](cli.ja.md)，`cascade_tree.png` の読み方は [可視化](visualization.ja.md) を参照．

```bash
cargo run --release -- run --platform x --n-agents 200 --n-leaders 20 --timesteps 30 --seed 42
uv run oasis-tools visualize
```

## 2. グループ極化の創発を見る

極化指数 `P = (1/N)·Σ(o_a − ō)²` は相互作用が進むにつれ増大傾向になるはず (意見はリポスト/いいねした内容へドリフトする)．`metrics_timeseries.png` のパネル 1．

## 3. RecSys アブレーション

推薦器を外し，情報拡散が阻害されること (伝播到達が低下し，スーパーユーザーのブロードキャストに限定される) を確認する:

```bash
cargo run --release -- run --recsys none --n-agents 200 --seed 42
```

## 4. スケール効果 (感度分析)

エージェント数 × 活性化率を走査し，極化と意見多様性が N とともに増大すること (論文の核心的主張) を確認する．`sweep_metrics_vs_n.png` は [可視化](visualization.ja.md) を参照．

```bash
cargo run --release -- sweep --n-agents-values 200,1000,5000 \
    --activation-rate-min 0.1 --activation-rate-max 0.5 --activation-rate-step 0.2 --runs 5
uv run oasis-tools visualize-sweep
```

## 5. 再現可能・オフラインの実行

プロンプトキャッシュにより再実行はコスト 0 で安定する．完全オフライン (LLM 通信なし) には mock スモークか `--n-leaders 0` を使う:

```bash
cargo run --release --example mock_smoke -- results
# または
cargo run --release -- run --n-leaders 0 --n-agents 40 --timesteps 10 --seed 42
```

決定論的コアが bit 再現的である一方 LLM レイヤがキャッシュで擬似決定論化される二層決定論は [アーキテクチャ](architecture.ja.md) を参照．
