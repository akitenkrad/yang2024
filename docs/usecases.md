# Use cases

What you can do with this OASIS reimplementation, and where to read more.

## 1. Observe an information-diffusion cascade

Run a small X simulation and watch a cascade spread over multiple stages (not a single broadcast). See [CLI](cli.md) for flags and [Visualization](visualization.md) for reading `cascade_tree.png`.

```bash
cargo run --release -- run --platform x --n-agents 200 --n-leaders 20 --timesteps 30 --seed 42
uv run oasis-tools visualize
```

## 2. Watch group polarization emerge

The polarization index `P = (1/N)·Σ(o_a − ō)²` should trend upward as interaction proceeds (opinions drift toward what agents repost/like). It is panel 1 of `metrics_timeseries.png`.

## 3. RecSys ablation

Remove the recommender and confirm information diffusion is impaired (propagation reach drops, limited to super-users' broadcasts):

```bash
cargo run --release -- run --recsys none --n-agents 200 --seed 42
```

## 4. Scale effect (sensitivity analysis)

Sweep agent count × activation rate and check that polarization and opinion diversity grow with N (the paper's core claim). See [Visualization](visualization.md) for `sweep_metrics_vs_n.png`.

```bash
cargo run --release -- sweep --n-agents-values 200,1000,5000 \
    --activation-rate-min 0.1 --activation-rate-max 0.5 --activation-rate-step 0.2 --runs 5
uv run oasis-tools visualize-sweep
```

## 5. Reproducible, offline runs

The prompt cache makes reruns free and stable. For a fully offline run (no LLM egress) use the mock smoke or `--n-leaders 0`:

```bash
cargo run --release --example mock_smoke -- results
# or
cargo run --release -- run --n-leaders 0 --n-agents 40 --timesteps 10 --seed 42
```

See [Architecture](architecture.md) for the two-layer determinism that makes the deterministic core bit-reproducible while the LLM layer is pseudo-determinised by the cache.
